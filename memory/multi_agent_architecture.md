# マルチエージェントアーキテクチャ設計

## 目的
- 処理速度の最大化
- マスターエージェントのトークン使用量の最小化
- フリーミアムモデルの活用によるコスト最適化

## アーキテクチャ

### エージェント構成
```
Master Agent (高機能モデル)
├── Collector Agent (フリーミアム)
├── Organizer Agent (フリーミアム)
├── Analyzer Agent (フリーミアム)
└── Reporter Agent (フリーミアム)
```

### 各エージェントの役割

#### Master Agent
- タスクの割り振り・調整
- 結果の統合・品質管理
- ユーザーとの対話
- トークン使用: 最小限（指示のみ）

#### Collector Agent
- inboxディレクトリの監視
- ファイル収集・メタデータ抽出
- 並列処理対応

#### Organizer Agent
- YAMLフロントマターの自動付与
- カテゴリ分類
- タグ付け

#### Analyzer Agent
- コンテキスト内容の分析
- 類似ファイルの検出
- 重複排除

#### Reporter Agent
- レポート生成
- Git操作の実行
- 進捗報告

## 並列処理フロー

1. **収集フェーズ**
   - Collectorがinboxをスキャン
   - 複数ファイルを並列で処理

2. **整理フェーズ**
   - Organizerが並列でYAML付与
   - カテゴリ分類を同時実行

3. **分析フェーズ**
   - Analyzerが類似ファイルを並列比較
   - 重複チェックを並列実行

4. **報告フェーズ**
   - ReporterがGit操作をまとめて実行
   - Masterに結果を報告

## トークン削減戦略

### Master Agent
- 結果の要約のみを受信
- 詳細なコンテキストはスレーブエージェントで処理
- ファイル内容は送信せず、メタデータのみやり取り

### スレーブエージェント
- フリーミアムモデルで実行
- 独立したタスクを並列処理
- 結果のみをMasterへ返却
